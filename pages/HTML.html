HTML - Week 1

Some things covered during 29/03/23 in-person lecture include:

  • The importance of and how to make a functional HTML 'boilerplate''
  • Head/Body tags
  • External/Internal links
  • Comments
  • Image reference links

HTML (Hyper Text Markup Language) is the building blocks of a webpage. An important thing to remember with HTML is that it is the way to get information onto a web page. It is not designed for making it look 'pretty' like CSS or interactive like JavaScript but it is the way to get the standard layout and contents onto the page.

We have covered some common things to look for when debugging such as:

  1. Spelling mistakes (more common in CSS)
  2. Incorrect file names/extensions
  3. Missing closing tags or '/'

In the Seminar 30/03/23 we recapped the above and also learnt some extra pieces of information, such as including a link to an email address and introducing 'favicons'. Please see HTML comments on index.html for how this went (please also see on Safari no favicon present).
